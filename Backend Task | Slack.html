There is a simplified unix type shell with the following features:

Commands

touch

This command takes a single argument which is a file path, and creates that file if it does not exist, otherwise does nothing.

Example

touch file1

will create a file called file1

touch dir/file1

will create a file called file1 inside the directory dir

mkdir

This command takes a single argument, which is a path for a directory, and will create the directory at the path if the parent directory exists, otherwise it will print parent directory does not exist.

Example:

mkdir dir

will create a directory dir in the current working directory.

mkdir noexist/dir

will print parent directory does not exist if the directory noexist does not exist in the current working directory.

cd

This command will change the current working directory. It will take a single argument which will be the path for the directory to switch to. It will print directory does not exist if a directory at the path does not exist.

Example:

cd dir

will change the current working directory to dir if it exists.

cd noexists

will print directory does not exist if the directory noexists does not exist in the current working directory.

pwd

This command will print the name of the current working directory. It takes no arguments. At the root directory, it will print nothing.

rm

This command will take one argument, which will be a path to a file or directory which needs to be removed, and it will remove that file, or the directory and all the files within it. If a file or directory does not exist at the given path, it will print file or directory does not exist.

ls

Prints a list of all files and directories in the current working directory. Directories are printed inside single quotes. For example, if the current directory has a file file1 and a directory dir, this command will print

file1
'dir'

echo

This command will have any number of arguments. Anything that follows it will be printed to the screen.

cat

This command will have any number of arguments, which will be paths to files, and it will concatenate the contents of those files and print them to the screen. If any file does not exist, it will print file does not exist instead.

Example:

Consider there is a file called file1 with contents hello and a file file2 with contents world in the current working directory

cat file1 file2

will print helloworld to the screen.

cat file1 noexist

will print file does not exist to the screen.

upper

This command takes one argument, which is a file path, and prints the contents of the file in uppercase. If the file does not exist, it will print file does not exist to the screen.

Example:

Consider there is a file file1 with the contents hello world in the current working directory.

upper file1

will print HELLO WORLD to the screen.

upper noexist

will print file does not exist to the screen.

lower

This command is similar to the upper command, but it will output lowercase instead of uppercase.

Input and Output Streams

Unix treats the screen and the keyboard as files, called stdout and stdin respectively. These special files can also be redirected to other files.

The > operator will redirect stdout to the file at the path that is after this operator. If the file does not exist, the file will be created. However, if the parent directory does not exist, parent directory does not exist will be printed instead.

For example:

echo hello > file1

will create a file called file1 in the current working directory with the contents hello.

echo hello > file1 > file2

will create two files, file1 and file2 in the current working directory, both with the contents hello.

The < operator will get contents from the file at the path that is after this operator, and put them in stdin. If a file does not exist at the specified path, file does not exist will be printed instead. This is the only way to get something inside stdin in our example, as we are not using the keyboard. All the commands which have at least one argument can read from stdin as well. If a command is given arguments in addition to stdin, then stdin is completely ignored, otherwise it is considered to be the first argument of the command (cat command will get stdin as first argument, other commands will get the contents of stdin as first argument). If this operator is used more than once, the contents of all the files are concatenated into stdin.

Example:

Consider there is a file called file1 in the current working directory with the contents hello world.

cat < file1

will print hello world to the screen.

echo < file1

will print hello world to the screen.

rm < file1

will remove the file or directory named hello world if it exists in the current working directory.

upper < file1 < file1

will print HELLO WORLDHELLO WORLD to the screen.

stdout for one command can also be piped as stdin to another command with the | operator.

Example:

echo hello | cat

will print hello to the screen.

echo file1 | rm

will remove a file called file1 from the current working directory if it exists.

If there is a file file1 with the contents hello and a file file2 with the contents world

cat file1 file2 | upper > file3

will create a file file3 in the current working directory with the contents HELLOWORLD.

Directory Structure

Initially, execution will start at the root directory which will have no name. All paths are relative to the current working directory and there are no absolute paths. A . is treated as the current directory, while a .. is treated as the parent to the current directory in the path. If the parent directory does not exist, for example, using cd .. when already at the root directory, nothing will happen. Therefore, if called from the root directory, mkdir a/../../b will create a directory b in the root directory, if directory a exists. Paths are separated by a /.
